# Our Solution: Sweep 🧹

We’re building tools for professional developers using JetBrains, starting with super fast autocomplete that’s smarter than Cursor Tab.\
\
We’ve fine-tuned our own LLM with a custom inference stack that suggests completions with sub-100ms latency. Unlike Cursor, we use the JetBrains code index to let our autocomplete model navigate your code like a real developer.

## Why we built Sweep

We originally started Sweep in 2023 to build an AI junior developer, which hit 7.6k stars on GitHub.

We soon realized that this was many years out. Instead, we decided to build a coding assistant that developers could use today. Kevin and I have been JetBrains users since we were in college, and we were surprised something like this didn’t exist yet.

We built Sweep for ourselves, and use Sweep Tab hundreds of times a day to write production-grade code instead of vibe coding. It’s boosted our productivity by 2x and we hope you like it too. \

# Problem

Current AI coding assistants have poor support for JetBrains. GitHub Copilot in JetBrains is a far cry from VSCode, and other plugins aren’t much better.\
\
We’ve heard:

_“I love the features in JetBrains as an IDE but the AI sucks”_

_“I see all of these AI coding assistants on VSCode, and I check if they support JetBrains, and I always get disappointed when they don’t.”_

Engineers who don’t use AI coding assistants are less productive. But engineers who have to switch away from their IDEs are even less productive.

# Solution

Sweep makes it easy to use AI in JetBrains with our self-hostable plugin. **Engineers keep their existing code editor with no data leakage and the** **best AI features**.\
\
We make it seamless to give the AI context. Sweep automatically sees the current file and codebase, saving you precious clicks.

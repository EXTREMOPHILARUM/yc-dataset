Hey everyone, we are [Severin](https://www.linkedin.com/in/b6b682ce450f9e1c8717c24cdb1c988a3ac717b6cbe6e1373a177119a742a434/), [Chris](https://www.linkedin.com/in/kindl/), and [Achilleas](https://www.linkedin.com/in/achilleas-mitrotasios/). Tremor started as an open-source UI library to create dashboards, which grew to more than 10K Github stars in less than 10 months. This is where we discovered how many people need to build consumer-friendly dashboards but lack the resources to do so.

**Problem**

Almost every SaaS requires visualizing data for their end-users. However, creating customer-facing dashboards from scratch requires a significant effort for both engineering and design teams.

Therefore, many companies cannot afford to invest resources into building a visualization layer, especially when analytics is not part of the core product. Often, this results in an underwhelming user experience and additional revenue left on the table.

**Solution**

The Tremor Platform is a visual dashboard builder made for developers. It allows users to connect to data sources, create dashboards with an easy-to-use interface, and embed them as a performant, white-labeled analytics layer into existing apps. For example, a bank could deploy a dashboard that visualizes spending to their customer portal, in days instead of months.

**Meet the team**

Severin, Achi and Chris (left to right) know each other since university. Severin and Chris worked at Accenture for 3+ years, building dashboards for Fortune 500 companies. Achilleas previously worked as a software engineer at Bloomberg. They started Tremor as a side project next to their full-time jobs as they could not find a satisfying solution for visualizing data on websites.

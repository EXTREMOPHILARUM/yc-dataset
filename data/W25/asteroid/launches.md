### **What is the problem?**

Startups need to integrate with external portals (carriers, suppliers, healthcare) but can’t wait months for APIs.

Many great browser-agent frameworks have emerged in the past year, but **non-technical teams still can’t build production-grade agents.**

Operations leaders, PMs, and founders are the ones who **understand the workflows and edge cases** and _should_ be the builders. Relying on developers to constantly fix prompts for every new edge case isn’t sustainable.

### **Solution: Astro - AI Builder for Complex Browser Agents**

Today we’re launching **Astro**, our AI builder that lets anyone create **complex, self-healing browser agents in under 2 minutes**. Tell Astro what you want, for example _“Build an agent to pull patient data from this EHR”_, and it generates the full agent.

![uploaded image](https://www.ycombinator.com/media/?type=post&id=96061&key=user_uploads/1651915/53714daf-535a-450a-8a76-82eb70d43007)

**Astro improves the agent using previous runs.** Behind the scenes, it writes code when needed, including Playwright scripts for deterministic steps, giving everyone code-level reliability.

![uploaded image](https://www.ycombinator.com/media/?type=post&id=96061&key=user_uploads/1651915/62627740-6f1e-47ce-a197-917e01ceade7)

Every part of the agent, including graphs, prompts, and model settings, is fully visible so you can make precise, surgical edits whenever needed.

### **How does it work**

1. **Build:** Non-technical teams create agents directly in the platform. Astro generates the prompts, agent graphs, and tools needed to handle even the most complex browser workflows.
2. **Deploy:** Developers drop in a single code snippet to call the agent through Asteroid’s API/SDKs.
3. **Supervise:** Monitor live sessions, review recordings, track performance, and trigger human-in-the-loop via Slack when needed.
4. **Improve:** Non-technical teams update prompts and ship new agent versions instantly as workflows change.

## Asteroid: Browser Agents with Human Oversight

**Asteroid makes browser automation seamless for regulated industries.** Not only developers but also non-technical teams can **build and run our browser agents with Asteroid**, turning 45-minute manual tasks into 2-minute automated processes.

Our browser agents can navigate legacy portals, fill out forms, and extract data—all without brittle scripts, high costs, or maintenance headaches.

![uploaded image](https://www.ycombinator.com/media/?type=post&id=88012&key=user_uploads/1651915/50a0915f-b720-4fff-923b-0b84e6017098)

### **The Problem**

Traditional automation tools like RPA are too expensive for non-enterprise customers, **costing $10,000 per automation**. Meanwhile, scaleups especially in high-risk industries like insurance or healthcare lack the resources to maintain Selenium or Playwright scripts, as these frequently break due to UI changes and the automations need to be extremely reliable.

### **The Solution**

Browser agents bridge this gap—offering the flexibility of scripting with the reliability of RPA. With 2025’s computer use models, **web automation is now 100x cheaper and more robust, unlocking thousands of new use cases.**

### **How It Works**

**Asteroid transforms any browser task into an agentic workflow.** Simply show Asteroid what to do—via a screen recording, SOP, or written instructions—and get an agentic workflow. You can then call it through our web UI or API.

Behind the scenes, Asteroid spins up a secure, headless browser, handles authentication, executes workflows, and seamlessly integrates human review when needed—all without burdening your developers. We ensure the agent improves over time.\

### **Highlighted Features**

🔹 **API-First Automation** – Trigger a browser agent in a web UI or with a single API call—no need for complex scripts or RPA setups.\
🔹 **Human-in-the-Loop** – Automate only where it’s safe, with built-in escalation for manual review when needed.\
🔹 **Intelligent Guardrails** – Set rules like “Never click this button” or “Ask before submitting” to prevent errors.\
🔹 **Continuous Learning** – Asteroid evaluates every execution, adapting to website changes and improving over time using evaluations and backtesting.

### **How We Got Here**

We’re both AI engineers who had to solve the same problem independently. Joe built AI systems that stopped $100M+ in fraud annually. David developed fraud investigation agents for fintechs.

After years of building mission-critical automation, we learned that automation only works with intelligent guardrails, human oversight, and continuous learning.

We first built Asteroid as a developer tool for safe AI deployment—offering guardrails, human-in-the-loop workflows, and continuous evaluations. But we quickly realized:

\*\*Browser agents need these safeguards even more.\*\*\
✅ **Guardrails** – "Don’t click on this."\
✅ **Human oversight** – "Ping me before you submit."\
✅ **Continuous learning** – "The website changed—adapt."

Instead of just developer tools, we’ve shifted to building browser agents with safety and reliability baked in!

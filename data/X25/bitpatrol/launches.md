### **The Problem**

Secret leaks are happening every day, impacting companies across every industry. Traditional scanning tools attempt to catch these leaks using rigid regex patterns and keyword matching.

![uploaded image](https://www.ycombinator.com/media/?type=post&id=90617&key=user_uploads/2475147/274a7869-42fd-41fc-86f4-668846d4d01e)

However, these methods struggle with secrets that don't follow set patterns, allowing critical leaks like these to slip through undetected.

![uploaded image](https://www.ycombinator.com/media/?type=post&id=90617&key=user_uploads/2475147/e67f96a4-a636-461e-9434-93a92f23f1ff)

### **Our Solution**

Unlike legacy tools that rely on regular expressions, BitPatrol uses our proprietary machine learning model to detect secrets based on code context and developer intent – not just string patterns.

![uploaded image](https://www.ycombinator.com/media/?type=post&id=90617&key=user_uploads/2475147/243b128a-4438-4e39-bf3b-7c6c1b8e3d32)

Once our model flags a potential secret, we reference it against our dataset of billions of public commits, docker images, and open source packages. This helps us rapidly assess the secret's exposure frequency, drastically reducing false positives and solving the issue of alert fatigue.

Using our [GitHub App](https://github.com/apps/bitpatrol), you can trigger actions that occur when a secret is detected, including automated alerts and custom workflows to immediately respond and remediate when a secret is leaked.

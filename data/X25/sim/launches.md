# Launches

## Sim Studio: UI to Build AI Agents

**_TL;DR_** [Sim Studio](https://www.simstudio.ai/) is an [open-source](https://github.com/simstudioai/sim) interface to build and deploy AI agent workflows.

![uploaded image](https://www.ycombinator.com/media/?type=post&id=90127&key=user_uploads/2320005/bcf7b4e1-6692-455c-932a-8f2047d2b652)

Our Figma-like canvas allows users to drag-and-drop connect LLMs to their favorite tools (Slack, Supabase, Pinecone, Gmail). Check out our launch video 👇

<https://youtu.be/EAAMNXRYncY>

**Key Features**

* 🤖 Build **Agents** that connect with any of your tools
* 🚀 **Deploy** your workflow as a chat or an API
* 💡 Quickly iterate and optimize your agents **visually**

**The Problem**

Agent frameworks require learning and extensive boilerplate code, and existing visual interfaces abstract most of the customization required for complex agent workflows.

**Our Solution**

![uploaded image](https://www.ycombinator.com/media/?type=post&id=90127&key=user_uploads/2320005/2d40df0f-fe2f-4c01-bbbb-1fa207339a81)

**Sim Studio** is a lightweight, user-friendly platform to build, test, and deploy AI agents in minutes. You can design the entire agent architecture, with a variety of models (hosted and local via Ollama), tools with granular tool use control, and structured output.

You can trigger the workflows manually, deploy as an **API**, or **schedule** the workflows to run periodically. They can also be set to trigger on incoming **webhooks** (e.g., [Slack messages](https://docs.simstudio.ai/tools/slack)) and deployed as standalone chat instances.
